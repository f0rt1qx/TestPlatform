# 👁️ Eye-Tracking для Test Platform

Полнофункциональная система отслеживания взгляда через веб-камеру во время прохождения тестов.

## 📋 Возможности

- ✅ **Калибровка** — автоматическая 9-точечная калибровка перед началом теста
- ✅ **Отслеживание взгляда** — непрерывный мониторинг направления взгляда
- ✅ **Фиксация** — определение точек фиксации взгляда (100ms+)
- ✅ **Логирование** — сохранение данных в базу каждые 5 секунд
- ✅ **Индикатор статуса** — визуальная обратная связь для пользователя
- ✅ **Graceful degradation** — если камера недоступна, тест продолжается без eye-tracking

## 🏗️ Архитектура

### Компоненты

1. **`public/js/eye-tracker.js`** — клиентский модуль eye-tracking
   - Использует библиотеку **WebGazer.js** (загружается с CDN)
   - Автоматическая калибровка с 9 точками
   - Обработка и фильтрация данных о фиксациях
   - Периодическая отправка данных на сервер

2. **`test.php`** — интеграция в интерфейс теста
   - Инициализация при загрузке теста
   - Остановка при завершении/дисквалификации
   - Логирование статуса в монитор активности

3. **`api/test.php`** — серверная часть
   - Разрешённый тип события: `eye_fixations`
   - Сохранение данных в таблицу `logs`

4. **База данных**
   - Таблица `logs` с типом события `eye_fixations`
   - JSON поле `event_data` содержит массив фиксаций

## 🚀 Использование

### Для пользователей

1. **Начало теста**
   - При открытии теста автоматически запрашивается доступ к камере
   - Появляется экран калибровки с 9 красными точками
   - Нужно нажать на каждую точку **5 раз**, глядя на неё
   - После калибровки появляется индикатор "Eye-tracking активен"

2. **Во время теста**
   - WebGazer непрерывно отслеживает взгляд
   - Данные о фиксациях сохраняются каждые 5 секунд
   - В логе мониторинга отображается статус eye-tracking

3. **Завершение теста**
   - Eye-tracking автоматически останавливается
   - Все данные отправляются на сервер

### Для администраторов

Данные eye-tracking сохраняются в таблице `logs`:

```sql
SELECT 
  l.attempt_id,
  l.event_type,
  l.event_data,
  l.created_at
FROM logs l
WHERE l.event_type = 'eye_fixations'
  AND l.attempt_id = 123
ORDER BY l.created_at;
```

#### Структура данных фиксаций

```json
{
  "fixations": [
    {
      "startX": 450.2,
      "startY": 320.5,
      "startTime": 1712239200000,
      "endX": 455.1,
      "endY": 318.9,
      "endTime": 1712239200250,
      "duration": 250,
      "points": 5
    }
  ],
  "count": 10
}
```

**Поля:**
- `startX`, `startY` — координаты начала фиксации (px от левого верхнего угла)
- `endX`, `endY` — координаты конца фиксации
- `startTime`, `endTime` — timestamp в миллисекундах
- `duration` — длительность фиксации в мс
- `points` — количество точек взгляда в фиксации
- `count` — общее количество фиксаций в пакете

## 📊 Анализ данных

### Примеры запросов

**Получить все фиксации для попытки:**
```sql
SELECT event_data, created_at
FROM logs
WHERE attempt_id = 123 
  AND event_type = 'eye_fixations'
ORDER BY created_at;
```

**Подсчитать общее количество фиксаций:**
```sql
SELECT SUM(JSON_EXTRACT(event_data, '$.count')) as total_fixations
FROM logs
WHERE attempt_id = 123 
  AND event_type = 'eye_fixations';
```

**Средняя длительность фиксации:**
```sql
SELECT 
  AVG(fix.duration) as avg_duration
FROM logs,
  JSON_TABLE(event_data, '$.fixations[*]' COLUMNS (
    duration INT PATH '$.duration'
  )) as fix
WHERE attempt_id = 123 
  AND event_type = 'eye_fixations';
```

### Визуализация

Для визуализации данных eye-tracking можно:

1. Экспортировать данные в CSV/JSON
2. Использовать Python/R для создания тепловых карт
3. Построить скан-пути (scanpaths)

Пример Python-скрипта для тепловой карты:

```python
import json
import matplotlib.pyplot as plt
from scipy.stats import gaussian_kde

# Загрузка данных из БД
fixations = []
for log in logs:
    data = json.loads(log['event_data'])
    for fix in data['fixations']:
        fixations.append((fix['startX'], fix['startY']))

x, y = zip(*fixations)

# Тепловая карта
xy = np.vstack([x, y])
z = gaussian_kde(xy)(xy)

plt.scatter(x, y, c=z, s=50, alpha=0.5)
plt.xlabel('X (px)')
plt.ylabel('Y (px)')
plt.title('Eye Gaze Heatmap')
plt.colorbar()
plt.show()
```

## 🔍 Применение

### Обнаружение списывания

- **Частые отвлечения взгляда** — много коротких фиксаций за пределами экрана теста
- **Фиксации на определённых областях** — возможно, смотрит на шпаргалки
- **Отсутствие фиксаций на вопросах** — пользователь не читает вопросы

### Анализ поведения

- **Время на чтение вопросов** — длительность фиксаций на тексте
- **Паттерны просмотра ответов** — как пользователь изучает варианты
- **Колебания взгляда** — уверенность/неуверенность в ответах

### Улучшение тестов

- **Сложные вопросы** — больше фиксаций и колебаний взгляда
- **Неоднозначные ответы** — частые переводы взгляда между вариантами
- **Оптимизация UI** — куда пользователи смотрят чаще всего

## ⚠️ Ограничения

1. **Точность** — WebGazer обеспечивает точность ~50-100px
2. **Освещение** — требуется хорошее освещение
3. **Положение камеры** — камера должна быть на уровне глаз
4. **Браузер** — работает только в Chrome/Firefox/Edge
5. **Разрешение** — требуется явное разрешение пользователя на доступ к камере

## 🔒 Конфиденциальность

- Данные камеры **не сохраняются** — обрабатываются только координаты взгляда
- Видео **не отправляется** на сервер — всё происходит локально
- Данные фиксаций **анонимизированы** — привязаны только к attempt_id
- Пользователь **должен дать согласие** на использование камеры

## 🐛 Решение проблем

### "Не удалось активировать отслеживание взгляда"

**Причины:**
- Нет доступа к камере
- Браузер блокирует доступ
- HTTPS требуется для некоторых браузеров

**Решение:**
- Разрешите доступ к камере в настройках браузера
- Убедитесь что используется `http://localhost` или HTTPS
- Проверьте что камера работает в других приложениях

### Калибровка не завершается

**Причины:**
- Пользователь не нажал на точки 5 раз
- WebGazer не загрузился

**Решение:**
- Убедитесь в стабильном интернет-соединении (для CDN)
- Следуйте инструкциям на экране калибровки

### Неточное отслеживание

**Причины:**
- Плохое освещение
- Калибровка выполнена некорректно
- Пользователь двигается

**Решение:**
- Улучшите освещение
- Перекалибруйте (перезагрузите тест)
- Сидите неподвижно во время теста

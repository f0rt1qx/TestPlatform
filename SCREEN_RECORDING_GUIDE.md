# 📹 Система записи экрана во время тестирования

## Обзор

Система записи экрана позволяет записывать экран студента во время прохождения теста для последующего анализа преподавателем. Это мощный инструмент для предотвращения мошенничества.

## Возможности

### ✅ Реализовано

1. **Запись экрана в реальном времени**
   - Используется MediaRecorder API браузера
   - Формат: WebM (VP8/VP9 кодек)
   - Качество: 2.5 Mbps
   - Максимальная длительность: 30 минут (настраивается)

2. **Chunked Upload с Retry-механизмом**
   - Автоматическая загрузка чанков во время записи
   - Размер чанка: 5MB
   - Retry при ошибках (3 попытки)
   - Таймаут запросов: 30 секунд для чанков, 60 секунд для финальной записи

3. **Индикатор записи в UI**
   - Красный индикатор "REC" в topbar во время записи
   - Пульсирующая анимация для привлечения внимания
   - Студент всегда знает, что идет запись

4. **Серверная валидация**
   - Проверка прав доступа (владелец попытки или админ)
   - Валидация размера файла (10MB для чанков, 100MB для финальной записи)
   - Проверка MIME типа и расширения файла
   - Логирование всех загрузок для аудита

5. **Админ-панель**
   - Новая вкладка "Записи" с просмотром всех записей
   - Отдельная страница `/recordings.php` для полноразмерного просмотра
   - Пагинация и статистика
   - Возможность удаления записей

## Как это работает

### Поток данных

```
1. Студент начинает тест
   ↓
2. Браузер запрашивает разрешение на запись экрана
   ↓
3. MediaRecorder начинает запись (1000ms timeslice)
   ↓
4. Данные накапливаются и загружаются чанками (~5MB)
   ↓
5. При завершении теста все оставшиеся данные загружаются как финальная запись
   ↓
6. Запись сохраняется на сервере и в БД
   ↓
7. Преподаватель может просмотреть запись из админ-панели
```

### Серверная часть

**Файл:** `api/recording.php`

Обработка POST-запросов:
- Валидация авторизации
- Проверка прав на попытку
- Валидация файла
- Сохранение через RecordingModel
- Логирование события

**Файл:** `api/recordings-list.php`

Обработка GET-запросов (только для админов):
- Список всех финальных записей
- Фильтрация по attempt_id, user_id, test_id
- Пагинация
- Статистика

**Файл:** `api/view-recording.php`

Потоковое видео с поддержкой HTTP Range requests (206 Partial Content).

## База данных

### Таблица: `recordings`

```sql
CREATE TABLE `recordings` (
  `id` INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  `attempt_id` INT UNSIGNED NOT NULL,
  `user_id` INT UNSIGNED NOT NULL,
  `file_path` VARCHAR(500) NOT NULL,
  `file_size` BIGINT UNSIGNED NOT NULL DEFAULT 0,
  `chunk_index` INT UNSIGNED NOT NULL DEFAULT 0,
  `is_final` TINYINT(1) NOT NULL DEFAULT 0,
  `duration` BIGINT UNSIGNED NOT NULL DEFAULT 0,
  `created_at` TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (`attempt_id`) REFERENCES `attempts`(`id`) ON DELETE CASCADE,
  FOREIGN KEY (`user_id`) REFERENCES `users`(`id`) ON DELETE CASCADE
);
```

## Безопасность

1. **Аутентификация**: Только авторизованные пользователи могут загружать записи
2. **Авторизация**: Только владелец попытки или админ может получить доступ
3. **Валидация файлов**:
   - Проверка MIME типа (video/webm, video/mp4)
   - Проверка расширения (.webm, .mp4)
   - Ограничение по размеру
   - Запрет пустых файлов
4. **Логирование**: Все загрузки логируются для аудита
5. **Удаление**: Только админы могут удалять записи

## Устранение неполадок

### Проблема: "Запись не начинается"

**Решение:**
1. Проверьте, что браузер поддерживает MediaRecorder API
2. Убедитесь, что студент дал разрешение на запись экрана
3. Проверьте консоль браузера на ошибки

### Проблема: "Chunk upload fails"

**Решение:**
1. Проверьте размер загружаемых файлов (логи в браузере)
2. Увеличьте `upload_max_filesize` и `post_max_size` в php.ini
3. Проверьте права на запись в `uploads/recordings/`

### Проблема: "Видео не воспроизводится"

**Решение:**
1. Убедитесь, что файл существует на сервере
2. Проверьте права доступа к файлу
3. Проверьте, что браузер поддерживает WebM формат

## Статистика и мониторинг

В админ-панели доступна статистика:
- Общее количество записей
- Общий размер всех записей
- Средняя длительность записи
- Количество неудачных загрузок
